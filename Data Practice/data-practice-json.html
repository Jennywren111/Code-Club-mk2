Now, here’s a few tasks!

The clue here is ‘understanding’. If you get to a point where you don’t finish any of these, but you simply understand how to do it, that’s good enough.

So below are some function names. I want you to make them work!

Tips

  • When it starts with log, I don’t care how - console, whatever, but just output stuff somewhere however you see fit
  • When it starts with ‘get’, then I want it to return ‘data’ of some sort. Don’t care what. An object, array, even a string!
  • When it starts with ‘find’ then I want it to find something based on the input.
  • Where applicable after each function name, there’s an example input. But obviously it should work with any valid value thrown at it.
  1. logNamesAndPhoneNumbers();
  2. getNamesAndPhoneNumbers();
  3. getEmail(index); // index = 5
  4. findEmail(uid); // uid = "51A2B622F"
  5. getCounty(town); // town = "Faringdon"
  6. logTownAndCounty(id); // id = 335
  7. getFullAddress(uid); // uid = "51A2B622F"
  8. getDrugUsed(firstName, surname); // firstName = "Ann", surname = "Smith"
  9. getParticipantEmails(drug); // drug = "control"
  10. getAverageEndTemperature(drug); // drug = "B"

Lastly, if you wish, you don’t have to code this, but think of ways you could write code to analyse this data to work out which drug was better/worse. As in maybe just come up with function names or ideas. 10, above, is a sort of clue!
